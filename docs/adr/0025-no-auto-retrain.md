# ADR-0025： 智能降噪不做自动回训(明确否决)

| 字段 | 内容 |
|------|------|
| **状态** | Accepted |
| **日期** | 2026-07-09 |
| **相关** | [ADR-0022](./0022-aiinsight-hitl-evidence.md)、[ADR-0012](./0012-triage-three-stage-pipeline.md) |

## 背景

智能降噪能从历史告警中学出"哪些是噪音",一个诱人的想法是让系统**自动回训**:无监督地自学习、持续更新模型或抑制规则,减少人工维护。本 ADR 记录对这条路线的**明确否决**,并锁定当前的人在环闭环。

## 决策

**不实现无监督自学习 / 模型回训。** 决策的核心是"不做什么":不让任何模型或规则在没有人确认的情况下自动改变自身行为。

当前保留的闭环(不变):

- `suggestNoise` 产出降噪**建议**。
- 人 **accept** 后,`applyNoiseSuggestion` 把建议沉淀为一条显式的 `SuppressionRule`。
- `/analytics/ai-feedback` 提供采纳率,供人工调优参考。

即:AI 只提议,规则的变更始终经过人确认这道闸,落地为可见、可审计的显式规则。

## 理由

- 自动回训会**自动改变模型 / 规则行为**,与"AI 带 evidence + HITL"基线(ADR-0022)直接冲突。
- 绕过人确认这道闸,可能把**真实告警误抑制**且**无人负责**——降噪出错的代价是"故障无人响应",不可接受。
- 显式 `SuppressionRule`(ADR-0012)可见可审计,自动回训则是黑箱漂移,与全项目的可溯源基线相悖。

## 备选方案

- **无监督自学习持续回训**:即本 ADR 否决的方案,理由如上。
- **未来若要做的前置条件**:须走独立评审;必须保留人的否决权;上线前先**影子运行**(shadow)、灰度验证,确认不会误抑真实告警后再逐步放量。在满足这些前置条件前不启动。

## 影响 / 权衡

- 降噪规则的维护仍需人工介入(accept 建议),牺牲了"全自动进化",换取"绝不误杀真告警且始终有人负责"。
- 采纳率数据(`/analytics/ai-feedback`)承担人工调优的信息支撑,替代自动回训的反馈回路。
- 该否决是可逆的:未来满足前置条件时可新增 ADR 取代本决策,决策演进史得以保留。
