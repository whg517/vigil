# 能力域 14-15：集成生态与数据报表

| 字段 | 内容 |
|------|------|
| **覆盖 PRD** | 能力域 14（集成）M14.1~M14.6、能力域 15（报表）M15.1~M15.6 |
| **文档版本** | v0.1 |
| **创建日期** | 2026-06-20 |
| **关联** | 各能力域（集成是横向打通）、能力域 12 复盘（报表度量） |

---

## 1. 目标

- **集成生态（能力域 14）**：让 Vigil 与外界**双向打通**——告警进得来、通知发得出、操作能联动、数据可编程访问。
- **数据报表（能力域 15）**：度量"做得怎么样"，用数据驱动改进。

---

## Part A：集成生态

### A1. 集成方向

| 方向 | 说明 | 对应 |
|------|------|------|
| **入向** | 告警源接入 | 能力域 1（webhook/邮件/API） |
| **双向** | IM 平台 | 能力域 8 |
| **出向** | 工单/通信/webhook | 工单系统、云语音、自定义 |
| **开放 API** | 程序化访问全部能力 | REST + WebSocket |

### A2. 出向集成

| 集成 | 用途 | 触发 |
|------|------|------|
| **工单系统**（Jira/禅道/自研） | 复盘 action_item 自动建工单 | 复盘发布 |
| **云语音**（阿里云/腾讯云） | 电话/SMS 通知 | 能力域 7 |
| **Webhook 出口** | Incident 生命周期事件推送外部 | 事件订阅 |

### A3. 开放 API（M14.4）

- **REST**：覆盖所有实体与操作（CRUD + 业务动作）。
- **WebSocket**：实时推送（事件状态、时间线、通知）。
- **API 版本**：`/api/v1/`，OpenAPI 描述契约。
- **鉴权**：API Key（scoped）。

### A4. Webhook 出口（M14.5）

用户可订阅 Incident 生命周期事件，Vigil 推送到自定义端点：

```json
POST {user_webhook_url}
{
  "event": "incident.resolved",
  "incident": { ... },
  "timestamp": "..."
}
```

### A5. 集成目录（M14.6）

内置常见集成的配置向导（如"接入 Prometheus"分步指引），降低配置门槛。

## Part B：数据报表

### B1. 度量维度

| 报表 | 指标 | 价值 |
|------|------|------|
| **告警度量**（M15.1） | 接入量、降噪率、unrouted 量 | 衡量分诊效果 |
| **事件度量**（M15.2） | Incident 数量、severity 分布、MTTA/MTTR | 衡量响应效率 |
| **团队负载**（M15.3） | 各团队值班次数、夜间打扰次数、人均事件数 | 衡量团队压力 |
| **复盘度量**（M15.4） | 复盘完成率、action_item 闭环率 | 衡量改进闭环 |
| **趋势**（M15.5） | 时间序列趋势，识别恶化/改善 | 衡量长期演进 |

### B2. 关键指标定义

| 指标 | 定义 |
|------|------|
| MTTA | Incident 创建到首次 ack 的平均时长 |
| MTTR | Incident 创建到 resolved 的平均时长 |
| 降噪率 | 1 - （触发通知的 Event / 原始 Event） |
| 夜间打扰次数 | quiet_hours 内通知值班人的次数 |

### B3. 数据来源

- 时序数据：从 Event/Incident/Action 表聚合。
- 聚合任务：定时 Asynq 任务（每小时/每日）。
- 实时看板：WebSocket 推送 + 前端聚合。

### B4. 导出（M15.6）

- CSV 导出。
- 对接 BI（通过开放 API / Webhook）。
